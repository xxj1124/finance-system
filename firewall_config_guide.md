# 阿里云轻量应用服务器防火墙配置详细步骤

## 注意事项

您使用的是**阿里云轻量应用服务器**，而不是ECS实例，因此在界面上没有"安全组"选项。在轻量应用服务器中，类似的功能叫做"**防火墙**"。

## 配置步骤

### 步骤1：进入防火墙页面

1. 在当前轻量应用服务器详情页顶部的导航栏中，找到并点击「**防火墙**」选项卡
2. 进入防火墙配置页面后，您会看到当前已有的防火墙规则列表

### 步骤2：开放22端口（SSH连接）

1. 在防火墙页面，点击右上角的「**添加规则**」按钮
2. 在弹出的「添加防火墙规则」窗口中，填写以下信息：
   - **应用类型**：选择「自定义」
   - **规则方向**：入站
   - **协议**：TCP
   - **端口范围**：22
   - **授权对象**：0.0.0.0/0
   - **备注**：SSH连接端口
3. 点击「**确定**」按钮，完成22端口的开放

### 步骤3：开放80端口（HTTP访问）

1. 再次点击「**添加规则**」按钮
2. 在弹出窗口中填写以下信息：
   - **应用类型**：选择「HTTP(80)」（可以直接选择预设的HTTP类型）
   - **规则方向**：入站
   - **协议**：TCP
   - **端口范围**：80
   - **授权对象**：0.0.0.0/0
   - **备注**：HTTP访问端口
3. 点击「**确定**」按钮，完成80端口的开放

### 步骤4：开放5000端口（应用访问）

1. 再次点击「**添加规则**」按钮
2. 在弹出窗口中填写以下信息：
   - **应用类型**：选择「自定义」
   - **规则方向**：入站
   - **协议**：TCP
   - **端口范围**：5000
   - **授权对象**：0.0.0.0/0
   - **备注**：Flask应用访问端口
3. 点击「**确定**」按钮，完成5000端口的开放

### 步骤5：验证防火墙规则

1. 返回防火墙规则列表页面
2. 检查是否已经添加了以下三条规则：
   - TCP 22端口
   - TCP 80端口
   - TCP 5000端口
3. 确保所有规则的「状态」都是「已启用」

## 防火墙配置完成

完成以上步骤后，您的轻量应用服务器已经开放了部署所需的所有端口：22、80和5000。

现在您可以返回部署指南，继续执行后续步骤了。
